## Фича: метр калибровки (CBM / Certainty-Based Marking) — PR #83
Ядро v2.9-«изюминки» портировано в `GillLearningSheet` → вкладку «Тест»:
- **Гейт уверенности:** варианты `disabled`; сначала выбор уверенности
  (`CONF=[Точно знаю d3, Кажется d2, Наугад d1]`) → `pickConf()` разблокирует.
- **Метр** `#glsQuizCalib`: `calScore/seen*100`%, тон `is-hi`≥70 (зелёный
  #3a9d5d) / `is-mid`≥40 (жёлтый #d4a017) / `is-lo`<40 (красный #c2603a).
- **Вердикт** `#glsQuizVerdict` 3 тона (по смыслу): good (совпадение) / warn
  (уверенная ошибка) / neutral (везение). `calScore++` только за good.
- Сохранены стрик, «Почему это важно», ссылка «Перечитать».
- Файлы: `GillLearningSheet.astro` (разметка+JS), `css/floating-cluster.css`
  (`.quiz-calib/.quiz-conf/.quiz-verdict`, скоуп `[data-gill-v16]`, токены
  `--gb-*` + `html.dark`-оверрайды для good/warn).

### Проверка (скрины @412px, реальные тапы)
- Гейт: `optsDisabledBeforeConf:true` во всех темах; после выбора — разблок.
- Прогон 6 ответов: метр 0→50→33→50→60→67%, классы `is-lo→is-mid`;
  вердикты `is-neutral` (наугад+верно), `is-good` (честно), `is-warn`
  (точно+неверно) — все рендерятся. День/сепия/ночь, 0 JS-ошибок.

## RC-04 (тач-зона скорости 44px) — НЕ сделано (осознанно)
Добавил `.mobile-speed::before{inset:-8px -2px}` (как в эталоне) → хит-тест
показал: вылет обрезается overflow рейла (`hitAbove` попадает в топ-бар, не в
пилюлю) → **мёртвый CSS**. Откатил. Чтобы заработало — рейлу нужен `min-height
~44px` (пилюли центрируются, у `::before` появляется место), но это риск микро-
прыжка бара при speed-open. Делать под присмотром, не ночью вслепую.
