## Контекст

Восстановление после упавшей сессии. Задача владельца: привести **мобильный вид
Гилла к эталонам 1:1** (или лучше), без «вольностей». Ключевой урок этого прохода:
чинить точечно = ходить по кругу. Правильный подход — **сверять весь экран с
каноническим референсом** (`references/gill-mobile/`), а не править «на глаз».

## Ключевые находки

- **«Баговання Германевтика» (крошки списком, бар за экраном) — НЕ баг сайта.**
  Это артефакт моей неполной сборки: `astro build` без шага
  `copy-legacy-to-dist.js` не кладёт `css/site.css`, `css/floating-cluster.css` в
  `dist` → глобальный CSS 404 → страница без стилей. **Для скринов только полная
  `npm run strangler:build:production-like`.**
- **part-TOC (оглавление части) — 3 бага в живом коде** (не мои правки): двойные
  рамки, «римские цифры» рендерятся точкой `·`, нет навигации между частями.
  Эталон v2.7 (`.btoc-*`, ~строки 609-660): пункт = только левая акцент-полоска
  (`border-left:2px`, скруглён справа), БЕЗ рамки-коробки; номер — мелкий справа,
  акцентный когда активен; есть прогресс-бар и шэр-футер; поиск = командная
  палитра `.cp-*`.
- **Аудит reader v5** (`references/gill-mobile/gill-reader-v5-AUDIT.md`): «Свободный»
  нужен `clamp` (не голый `nowrap`); дубли цитат в `js/highlights.js` (нет
  дедуп-гарда перед `unshift`); «скрепка у Настройки» = артефакт вебвью, не наш код.
