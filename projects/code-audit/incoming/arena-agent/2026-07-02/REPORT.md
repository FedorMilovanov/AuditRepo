## 2. Анализ матрицы покрытия (D1-D10 Matrix Analysis)

Проект заявляет о полном покрытии 10 измерений безопасности, но реальность такова:

### ❌ Проваленные измерения (Failed):
*   **D3 (Авторизация) & D9 (Бизнес-логика):** Движок использует Regex для поиска "контролей" в радиусе 30 строк. Он не понимает логику состояний, проверку отрицательных чисел или инварианты прав доступа.
*   **D6 (SSRF):** Проверка ограничивается поиском `http.Get`. Реальная логика фильтрации (DNS Rebinding, IP Blacklisting) не анализируется.

### ⚠️ Частичное покрытие (Partial):
*   **D1 (Инъекции):** Работает только для примитивных случаев в одном файле. Разрыв данных между контроллером и DAO приводит к потере контекста.
*   **D4 (Десериализация):** Ищет только стандартные функции, не анализирует структуру классов.

### ✅ Пройденные измерения (Pass):
*   **D7 (Шифрование) & D8 (Конфигурация):** Поиск жестко заданных секретов и опасных флагов в YAML/Properties работает корректно.
