## 1. Executive Summary

A structural check of exported SVG assets and sub-basemaps highlights two additional quality failures:

1. **Unescaped HTML Entity Errors in Exported SVGs (`SVG-P1-01`):**
   - Exported standalone SVG files (`images/atlas-export/avraam.svg` and `images/atlas-export/shvatim.svg`) generated by `scripts/atlas-export-sheet.js` contain raw `&nbsp;` HTML entities in text labels (lines 425, 427, 445, 530).
   - Standard XML tools and image pipelines fail to parse these files with `ERROR undefined entity: line 425, column 93` because `&nbsp;` is not a predefined entity in standalone XML/SVG.

2. **Extremely Coarse Low-Vertex Geography in Regional Sub-Basemaps (`BASE-P2-01`):**
   - `base-geo-mediterranean.svg` covers the entire Mediterranean sea basin (Italy, Greece, Asia Minor, Egypt, Levant) with only 12 path elements and 123 total SVG commands (~10 commands per coastline).
   - `base-geo-urheimat.svg` covers the Ancient Near East from Ararat to the Persian Gulf with only 68 total SVG commands, resulting in simplified, blocky terrain outlines.

3. **Absence of Curved Path Definitions Across 10/11 Maps (`DATA-P2-01`):**
   - Out of 11 map routes, only `avraam/route.json` declares curved trade route paths (`stages[].paths`).
   - The remaining 10 routes (`ishod`, `pavel`, `revelation`, `shvatim`, `shoftim`, `melachim`, `yeshua`, `maccabim`, `early-church`, `nachalo`) contain zero curved path definitions, forcing straight-line node graphs even if `map-engine.js` path parsing is fixed.
