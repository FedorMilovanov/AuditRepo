## Ключевое открытие: системная проблема dark-theme ремапа

Предыдущий аудит выявил 12 багов. Углублённый анализ обнаруживает **единую корневую причину** большинства визуальных проблем: **dark-theme ремап в `mobile-hotfix.css` покрывает только 800/900 уровни Tailwind-цветов, оставляя 500/600/700 без покрытия**. Это создаёт три класса проблем одновременно:

2. **Потеря идентичности** — средние цвета (600) остаются дефолтными Tailwind вместо теплых акцентных
3. **Несогласованные бордеры/фоны** — 54 Tailwind-класса не ремаппятся

## NG-DARK-01 (P1 · корневая): Неполный dark-theme ремап — 54 Tailwind-класса без покрытия

**Статус:** корневая причина для NG-VIS-01, NG-VIS-07, NG-VIS-08 из предыдущего отчёта

### Что ремаппится (работает)

| Уровень | Классы | Результат в dark |
|---|---|---|
| -900 | blue/emerald/purple/rose/amber/red/stone/teal | `#f3eadb` (яркий тёплый белый) |
| -800 | blue/emerald/purple/rose/amber/indigo/stone/teal | `rgba(238,231,220,.78)` (приглушённый) |
| -200/-300 | amber/emerald/purple | `rgba(238,231,220,.78)` |
| stone-500/600 | — | `rgba(238,231,220,.78)` |
| stone-700/800/900 | — | `#f3eadb` |

### Что НЕ ремаппится (ломается)

| Уровень | Использований | Проблема |
|---|---|---|
| `text-{accent}-600` | **165 раз** | Теряет идентичность главы — Tailwind дефолт вместо тёплого акцента |
| `text-{accent}-700` | **75 раз** | **Невидимый текст** — тёмный цвет на тёмном фоне |
| `text-{accent}-500` | **5 раз** | Низкий контраст |
| `text-{accent}-800` | **7 раз** (amber-800 в ch.4) | Не попадает в ремап (только blue/emerald/purple/red/rose/indigo/teal — **не amber**) |
| `bg-rose-50` | **13 раз** (ch.5) | Светло-розовый → не адаптирован к dark |
| `bg-stone-100/200` | **5 раз** | Светло-серый → не адаптирован к dark |
| `border-stone-100` | **52 раза** | Светлая рамка → невидимая в dark |
| `border-rose-100/200/400` | **8 раз** (ch.5) | Не адаптированы |
| `border-stone-300/700` | **6 раз** | Не адаптированы |

### Per-chapter breakdown: акцентные цвета без dark-ремапа

| Глава | Акцент | `text-{accent}-600` | `text-{accent}-700` | Дефолт Tailwind (dark) | Должно быть |
|---|---|---|---|---|---|
| I | blue | 40× | 20× | `#2563eb` (сырой синий) | тёплый приглушённый синий |
| II | emerald | 11× | 10× | `#059669` (низкий контраст) | тёплый приглушённый зелёный |
| III | purple | 40× | 12× | `#9333ea` (сырой фиолетовый) | тёплый приглушённый фиолетовый |
| IV | amber | 33× | 4× | `#d97706` (сырой оранжевый) | тёплый золотистый |
| V | rose | 41× | 1× | `#e11d48` (сырой розовый) | тёплый приглушённый розовый |

### Профессиональное решение

Вместо продолжения паттерна `!important` blanket-override (который уже содержит 12 групповых селекторов и 3 отдельных `!important`), нужно **перейти на CSS custom properties per-chapter**:

```css
/* Замена blanket-override на chapter-scoped переменные */
body.nagornaya-page {
  --ng-accent: var(--color-accent);
  --ng-accent-soft: var(--color-accent-soft);
  --ng-accent-strong: var(--color-accent-strong);
}

/* Per-chapter override — только акцент, остальное наследует */
body.nagornaya-page[data-chapter="1"] { --ng-accent: #2563eb; --ng-accent-soft: #eff6ff; }
body.nagornaya-page[data-chapter="2"] { --ng-accent: #059669; --ng-accent-soft: #ecfdf5; }
body.nagornaya-page[data-chapter="3"] { --ng-accent: #9333ea; --ng-accent-soft: #faf5ff; }
body.nagornaya-page[data-chapter="4"] { --ng-accent: #d97706; --ng-accent-soft: #fffbeb; }
body.nagornaya-page[data-chapter="5"] { --ng-accent: #e11d48; --ng-accent-soft: #fff1f2; }

html.dark body.nagornaya-page[data-chapter="1"] { --ng-accent: #93c5fd; --ng-accent-soft: rgba(147,197,253,.12); }
html.dark body.nagornaya-page[data-chapter="2"] { --ng-accent: #6ee7b7; --ng-accent-soft: rgba(110,231,183,.12); }
html.dark body.nagornaya-page[data-chapter="3"] { --ng-accent: #c4b5fd; --ng-accent-soft: rgba(196,181,253,.12); }
html.dark body.nagornaya-page[data-chapter="4"] { --ng-accent: #fcd34d; --ng-accent-soft: rgba(252,211,77,.12); }
html.dark body.nagornaya-page[data-chapter="5"] { --ng-accent: #fda4af; --ng-accent-soft: rgba(253,164,175,.12); }
```

Это позволяет:
1. Каждой главе — уникальный акцентный цвет в dark-теме (не одинаковый warm-gray)
2. Заменить 54 Tailwind-класса на `var(--ng-accent)` / `var(--ng-accent-soft)`
3. Удалить blanket `!important` override из `mobile-hotfix.css`
4. Автоматически покрывать все уровни (500/600/700/800/900)

**Требует:** добавления `data-chapter="N"` на `<body>` через Astro-компоненты каждой части.

## NG-DARK-02 (P1): `text-amber-800` не покрывается ремапом

**Затронуто:** chast-4 (7 использований)

Ремап `text-*-800` покрывает: blue, emerald, purple, red, rose, indigo, teal, stone — **но НЕ amber**. В ч.4 `text-amber-800` (#92400e) на `bg-white`/`bg-amber-50` в тёмной теме → очень тёмный текст на тёмном фоне.

**Использование:** заголовки типа «Богодухновенность (inspiration)», «Непогрешимость», «Concursus», «Ipsissima verba» — ключевые термины.

**Профессиональное решение:** то же что NG-DARK-01 — через `var(--ng-accent)`. Как interim: добавить `text-amber-800` в существующий -800 групповой селектор.

## NG-DARK-03 (P1): `border-stone-100` — 52 невидимых рамки в dark-теме

**Затронуто:** все 5 частей, почти каждая секция

`border-stone-100` (#f5f5f4) — очень светлая рамка на белом фоне. В dark-теме `bg-white` → `#151922`, но `border-stone-100` остаётся #f5f5f4 — **яркая белая рамка на тёмном фоне**, инвертированный визуальный эффект (должна быть тёмная, а стала яркой).

**Профессиональное решение:** ремап `border-stone-100` → `var(--color-border)` или `rgba(242,200,121,.15)` (тёплая тонкая рамка), вместе с рефакторингом на `var(--ng-accent-soft)` для акцентных рамок.

## NG-INLINE-01 (P1): «Из библиотеки» — полный рефакторинг

Обновляю NG-VIS-01/09/11 — все три являются гранями одной проблемы.

### Текущее состояние

5 файлов SectionX (по одному на часть) содержат идентичный блок «Из библиотеки» ~2KB на inline-стилях:

```
style="display:flex;flex-direction:column;gap:10px;padding:1.1rem 1.25rem;background:#faf8f5;
border:1px solid rgba(184,136,42,.2);border-radius:10px;..."
style="font-family:monospace;font-size:9px;letter-spacing:.1em;text-transform:uppercase;color:#b8882a;..."
style="font-size:.93rem;font-weight:600;color:#1c1410;..."
style="font-size:.76rem;color:#8a7968;..."
```

Проблемы:
- `color:#1c1410` — невидим в dark
- `color:#8a7968` — еле видим в dark
- `color:#b8882a` — не адаптирован
- `background:#faf8f5` — светло-розовый в dark
- Код дублируется 5×

### Профессиональное решение

1. **Создать Astro-компонент** `NagornayaLibraryLinks.astro` с Tailwind-классами:
   - `bg-[var(--color-surface)]` вместо `background:#faf8f5`
   - `text-[var(--color-text-primary)]` вместо `color:#1c1410`
   - `text-[var(--color-text-muted)]` вместо `color:#8a7968`
   - `text-[var(--color-accent)]` вместо `color:#b8882a`
   - `border-[var(--color-border)]` вместо `rgba(184,136,42,.2)`

2. **Импортировать** во все 5 SectionX компонентов

3. **Данные** — вынести ссылки в JSON/YAML в `data/nagornaya-library-links.json`

Это автоматически решает NG-VIS-01, NG-VIS-09, NG-VIS-11.

## NG-REVEAL-01 (P2): Мёртвый `reveal` — уточнение

Обновляю NG-VIS-05. Проверка dist-рендера подтверждает: класс `reveal` используется в 12 местах (ch.4 — 1×, ch.5 — 11×), но CSS-правила нет.

Глоссарий в `js/glossary.js` ищет `div.reveal` для гидратации терминов — так что `reveal` используется **для целей глоссария, не для анимации**. Класс `reveal` служит контейнером, внутри которого glossary.js ищет `<p>` для разметки.

**Но:** в `site.css` правило `body.gbs-world .reveal{opacity:1;transform:none;filter:none}` намекает, что на страницах Гилла `reveal` когда-то был анимирован, и это правило «отключает» анимацию. На страницах Нагорной **анимации никогда не было** — класс используется только как семантический маркер для глоссария.

**Профессиональное решение:** 
- Оставить `reveal` для глоссария (он функционально нужен)
- Если планируется scroll-reveal — добавить CSS-правило с `IntersectionObserver` (не через класс, а через `data-reveal` атрибут для разделения ответственности)
