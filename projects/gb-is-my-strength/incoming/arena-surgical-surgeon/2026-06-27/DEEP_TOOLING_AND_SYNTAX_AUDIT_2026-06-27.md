## 2. Скрытый баг "Поглощения Списка" в `scripts/download-fonts.js` (Syntax Swallowing Bug)

При запуске команды `npm run fonts:download` консоль выдавала успешный лог и завершалась с кодом `0`:
```text
  ✓ undefined (11480 bytes)
  skip exists: Lora/lora-cyrillic-500
  ...
Готово.
```

### 2.1 Вскрытие исходного кода (`scripts/download-fonts.js`, массив `SPECS`):
Давайте посмотрим на строки 10-18 в исходном коде скрипта:

```javascript
const SPECS = [
  ['Lora', 400, 'normal',  'cyrillic', 'Lora/lora-cyrillic-400'
  ['Noto Serif Hebrew', 400, 'normal', 'hebrew', 'NotoSerifHebrew/notoserifhebrew-400'],
  ['Noto Serif Hebrew', 500, 'normal', 'hebrew', 'NotoSerifHebrew/notoserifhebrew-500'],
],
  ['Lora', 500, 'normal',  'cyrillic', 'Lora/lora-cyrillic-500'],
```

### 2.2 Физиология синтаксической ошибки в JS
Обратите внимание на первую строку:
`['Lora', 400, 'normal',  'cyrillic', 'Lora/lora-cyrillic-400'` — **в конце строки отсутствует закрывающая квадратная скобка `]` и запятая!**

Из-за этой опечатки интерпретатор V8 (Node.js) разбирает этот блок не как список из трех независимых массивов, а как один непрерывный выражение-индекс (String Subscript / Property Accessor)!

Для Node.js этот код эквивалентен следующему:
```javascript
const stringValue = 'Lora/lora-cyrillic-400';
const indexValue = ['Noto Serif Hebrew', 400, 'normal', 'hebrew', 'NotoSerifHebrew/notoserifhebrew-400']; // превращается в строку при обращении

const result = stringValue[indexValue]; // 'Lora/lora-cyrillic-400'['Noto Serif...'] -> undefined
```

В результате:
1.  Первый элемент массива `SPECS` превращается в:  
    `['Lora', 400, 'normal', 'cyrillic', undefined]`.
2.  Спецификации для шрифтов **`Noto Serif Hebrew` (регулярный и медиум) полностью поглощаются и выбрасываются интерпретатором** как часть индексного выражения! Скрипт даже не пытается их обработать или проверить.
3.  При скачивании первого шрифта (Lora-400) имя файла вычисляется как `dest = undefined`. Скрипт успешно скачивает шрифт с серверов Google и сохраняет его на диск по адресу:  
    📂 **`fonts/undefined.woff2`**!
4.  Мы проверили дисковую систему песочницы: этот файл действительно физически существует в папке `fonts/` и имеет размер 11480 байт!

### 2.3 Почему это не ловило CI и ворота сборки?
*   Потому что синтаксически это **валидный JavaScript** (обращение к свойству строки через массив допустимо, хоть и возвращает `undefined`). Линтеры без строгих правил типизации пропускали этот код.
*   Шрифты `notoserifhebrew-400.woff2` и `notoserifhebrew-500.woff2` уже лежали в репозитории закомиченными ранее, поэтому при локальном рендере в браузере иконки иврита отображались правильно, маскируя тот факт, что скрипт загрузки для них сломан.

## 3. Сравнение валидаторов JSON-LD: Скрытый риск деплоя

Мы сопоставили два механизма валидации JSON-LD в CI пайплайне:
1.  Канонический скрипт: `scripts/dist-jsonld-audit.js`.
2.  Внедренный inline-скрипт в `.github/workflows/deploy.yml` (шаг "Dist JSON-LD parse audit").

### Результаты сопоставления:
*   **Регулярное выражение канонического скрипта:**  
    `/<script\s+type=["']application\/ld\+json["'][^>]*>([\s\S]*?)<\/script>/gi`  
    *Строго и безопасно:* парсит только реальные теги `application/ld+json`, поддерживает любые кавычки и пробелы.
*   **Регулярное выражение inline-скрипта в YAML:**  
    `/<script[^>]+application\\/ld\\+json[^>]*>([\\s\\S]*?)<\\/script>/g`  
    *Грубо и потенциально уязвимо:* может случайно среагировать на нецелевой тег `<script>`, если строка `application/ld+json` встретится внутри других атрибутов или комментариев.
*   **Строгость проверок:**  
    Канонический скрипт умеет выявлять и логировать **пустые блоки JSON-LD** и выводить отформатированные сниппеты сломанного JSON в консоль для мгновенной отладки. Inline-скрипт просто падает без подробностей.

**Рекомендация:**  
Удалить дублирующий шаг в `deploy.yml` и оставить только `npm run dist:jsonld:audit`, предварительно исправив регулярное выражение в `check-workflows.js` ( playbook Шаг 1). Это сделает проверку более строгой, профессиональной и уберет "костыли" из CI.

## 4. Резюме хирургического самоанализа

Мы обнаружили не просто косметическую опечатку, а **классический "молчаливый баг" (silent bug)**, который:
1.  Создавал мусорный файл `fonts/undefined.woff2` при каждой чистой сборке.
2.  Полностью отключал автоматическое обновление шрифтов иврита (`Noto Serif Hebrew`), необходимых для рендеринга генеалогического древа и теологических цитат.
3.  Успешно завершался с кодом `0`, обманывая все защитные системы.

Исправление этой опечатки (добавление `]` в конце 10-й строки `download-fonts.js`) — микроскопический, но крайне важный шаг для гигиены кодовой базы.
