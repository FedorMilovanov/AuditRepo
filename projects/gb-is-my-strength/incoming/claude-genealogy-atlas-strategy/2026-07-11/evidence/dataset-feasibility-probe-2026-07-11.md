# Evidence — практическая проверка датасетов (скачаны и распарсены, 2026-07-11)

Оба кандидата-ядра СКАЧАНЫ и проверены реальным парсингом (не по описаниям).
Команды: curl с raw.githubusercontent.com + python3-парсеры (см. commands.md).

## 1. Theographic Bible Metadata — people.json (5 072 606 байт)

URL: `https://raw.githubusercontent.com/robertrouse/theographic-bible-metadata/master/json/people.json`

**3 067 записей персон.** Покрытие полей (записей с полем):

| Поле | Записей | Комментарий |
|---|---|---|
| personLookup / name / gender / verses / verseCount / slug | 3 067 | 100% — у каждого все стихи упоминания |
| minYear / maxYear | 3 067 | грубая датировка |
| eastons (словарная статья) | 1 816 | биографический текст (Easton's, EN) |
| **father** | **1 584** | 52% — остальные упомянуты без родословной связи |
| children | 963 | |
| siblings | 944 | |
| memberOf (группы/народы) | 736 | готовый вход для кластеров |
| alsoCalled | 351 | альт-имена |
| mother | 200 | |
| partners | 173 | |
| birthYear/deathYear/birthPlace/deathPlace | 45–77 | редко |

Формат: Airtable-экспорт (`id: rec…`, `fields: {…}`), связи — массивы rec-ID.
Вывод: полнота имён высокая (3 067), связность средняя (52% с отцом) — то, что
видел прецедент bible-family-tree («disconnected components»). Лицензия CC BY-SA 4.0
⇒ используем как НЕЗАВИСИМОГО СВИДЕТЕЛЯ для сверки, в производный датасет не копируем.

## 2. STEPBible TIPNR (8 611 754 байт)

URL: `https://raw.githubusercontent.com/STEPBible/STEPBible-Data/master/Proper%20Nouns/TIPNR%20-%20Translators%20Individualised%20Proper%20Names%20with%20all%20References%20-%20STEPBible.org%20CC%20BY.txt`

Структура: секции `$====PERSON(s) / PLACE(s) / OTHER(s)`; записи разделены `$`,
поля — табами. **Всего $-записей: 4 269; топ-строк Name@Book.c.v: 4 233.**
Поля персоны: UnifiedName═uStrong · Description · **Parents (Male+Female одной
колонкой: `Amram@… + Jochebed@…`)** · Siblings · Partners · Offspring ·
Tribe/Nation of father · #Summary (с ref-разметкой) · Gender.

Грубые счётчики по строкам с ≥9 полями (уточнит Phase 1 секционным парсером):
- **с заполненными Parents: 3 329**
- с Offspring: 2 047
- с Partners: 1 249

Пример (Aaron): родители `Amram@Exo.6.18-1Ch + Jochebed@Exo.6.20-Num`, сиблинги
Моисей+Мариам, партнёр Elisheba, 4 сына, `Tribe of Levi`, summary с
`<ref="Exo.4.14">`. Маркеры данных: `(?)` — решение при неоднозначности
(документируются на TinyURL.com/STEP-GenealogyDecisions), `(d)` — народ-потомок,
`(a)` — предок вместо родителя, `(f)` — основатель. Есть официальные JSON-выгрузки
(People/Places/Others, Google Drive-ссылки в шапке файла).

Вывод: **связность TIPNR (~3.3k записей с родителями) существенно богаче
Theographic (1 584 отца)** + документированные генеалогические решения + иврит/греч
формы + все ссылки. Подтверждён как ядро пайплайна v2.

## 4. Следствия для REPORT §S5.1 (уточнение цифр)

- Реальный целевой объём: **~3.0–3.4k именованных персон** (Theographic 3 067;
  TIPNR-персон до ~3.3–4.2k включая (?)-варианты — точное число даст секционный
  парсер Phase 1). Иллюстративное «3 254» с референса оказалось в реальном диапазоне.
- Связей parent-типа ожидаем ~3.3–5k + spouse ~1.2k (реф. «7 856 связей» достижим
  только с учётом sibling/membership-рёбер — так и считать в счётчике страницы).
- «Скрыть пустые» из референса = фильтр персон без связей (у Theographic таких ~48%).

## 5. Микро-эксперимент: извлечение русских имён из Синодального текста (ДОКАЗАНО)

Полный Синодальный JSON скачан (thiagobodruk/bible `ru_synodal.json`, 5 924 506 байт,
66 книг; сам перевод — public domain). Секционный парсер TIPNR:

- **PERSON-секция: 3 056 top-записей** (М/Ж) — сходится с Theographic (3 067):
  реальный универсум именованных персон Библии ≈ 3.0–3.1k. Иллюстративное «3 254»
  с референса — в 6% от реальности.
- **Резолв стихов первого упоминания: 60/60** случайной выборки TIPNR-рефов
  (Book.ch.v) нашлись в Синодальном JSON (мэппинг OSIS-код → индекс книги).
- **Ручная проверка выравнивания (9 персон):** Peleg→«Фалек» (Быт 10:25 «имя одному:
  Фалек»), Seth→«Сиф», Enosh→«Енос», Aaron→«Аарон», Zerubbabel→«Зоровавель»,
  Boaz→«Вооз», Methuselah→«Мафусал» (Быт 5:21 «Енох… родил Мафусала»), Rahab→«Раав»
  — **8/9 ✓**. Промах: Jesse@Rut.4.17 → JSON вернул текст Руф 4:18 («Фарес родил
  Есрома») — сдвиг версификации ESV↔Синодальный в конце Руф 4.
- Вывод: механическое извлечение русских имён по стиху первого упоминания —
  РАБОТАЕТ на ~90%+ прямо «в лоб»; хвост закрывается таблицей версификации
  (у STEPBible в репо есть папка `Versification` — родные данные для этого) +
  паттернами («родил X», «имя одному: X», списки «сыновья Y: A, B, C») + очередь
  редактуры. Главный риск плана (ru-номенклатура ~3k имён) понижен с «Высокая» до
  «Средняя»: базовый слой добывается автоматически, редактура — выборочная сверка.

Команды — commands.md §probe-2.

## 6. Графовый зонд TIPNR (черновой парсер — цифры индикативные, нижние оценки)

Наивный парсер родительской колонки (Phase 1 обязан написать строгий: "+"-склейка
отец+мать, маркеры (a)/(d)/(f), хвосты -Book):

- Разрешено родительских рёбер: 1 847 (из ~3.3k строк с родителями — остальное
  потерял черновой нормализатор ключей; НЕ факт данных, факт парсера).
- **Иисус → Адам ДОСТИЖИМ по родительским рёбрам: 73 поколения (min)** —
  канонический счёт по Луке ≈ 76: золотой хребет выводится из TIPNR напрямую.
- Предков Иисуса в графе: 101 (Мф+Лк ветви).
- Крупнейшая связная компонента: 964 персоны; изолированных (при черновом
  парсере): ~1 043 — порядок совпадает с Theographic (~48% без отца) ⇒ фильтр
  «Скрыть пустые» и дизайн «изолированные персоны в кластерах-списках» обязательны.
- **Обнаружен цикл (self-loop Mamre@Gen.14.13)** — артефакт чернового парсера
  (Мамре — человек/место + (f)-маркер), но вывод общий: валидатор циклов
  родительского графа в пайплайне — доказанно необходим, а не «на всякий случай».

Все цифры §6 — L0-индикаторы для проектирования; канонические числа даст строгий
парсер Phase 1.
