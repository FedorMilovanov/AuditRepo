## 1. New Findings (Pass 13)

### NEW-41 [P2] — Stale sitemap.xml: 8 Production Routes Missing

- **Title:** Committed sitemap.xml is missing 8 production routes (holding pages + karty)
- **Severity:** P2 (High — SEO)
- **Source file(s):** `sitemap.xml`
- **Evidence:**
  ```bash
  # Sitemap has 42 URLs
  # page-ownership has 52 production routes (excl /dev/astro-test)
  # 10 routes not in sitemap:
  
  # Intentional exclusions (2):
  /izbrannoe  → noindex, localStorage-only page (filtered in astro.config.mjs)
  /konfessii/russkij-baptizm/_app  → built-app, internal React app
  
  # Unintentional (8) — production pages NOT in sitemap:
  /karty/early-church       → Astro page, production-dist, no noindex
  /karty/maccabim           → Astro holding page, production-dist, no noindex
  /karty/melachim           → Astro holding page, production-dist, no noindex
  /karty/pavel              → Astro holding page, production-dist, no noindex
  /karty/revelation         → Astro holding page, production-dist, no noindex
  /karty/shoftim            → Astro holding page, production-dist, no noindex
  /karty/shvatim            → Astro holding page, production-dist, no noindex
  /karty/yeshua             → Astro holding page, production-dist, no noindex
  ```
- **Root cause:** sitemap.xml is a LEGACY committed file (not auto-generated by Astro at deploy). copy-legacy-to-dist.js explicitly removes Astro-generated sitemap partials and keeps the legacy one. When new Astro pages were added, the legacy sitemap was not updated.
- **Impact:** Search engines can't discover 8 pages via sitemap. While internal links may still lead to these pages, sitemap omission reduces crawl priority and discovery speed.
- **Recommendation:** Either (a) update sitemap.xml to include all production-dist routes, or (b) switch to Astro-generated sitemap with proper filtering (requires migrating the sitemap filter to exclude /izbrannoe, /karty/early-church, and _app).
- **Confidence:** high
- **Verification level:** L2
- **Suggested repair lane:** `lane/seo-sitemap`

## 3. Strangler Build Architecture (Verified)

```
┌─────────────────────────────────────────────────────┐
│  astro:build                                        │
│  → dist/ (Astro pages only)                         │
│  → sitemap-index.xml, sitemap-0.xml (partials)      │
└─────────────┬───────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────────┐
│  copy-legacy-to-dist.js                             │
│  → copies legacy HTML (respects page-ownership)     │
│  → copies public root files (sitemap.xml, etc.)     │
│  → REMOVES Astro-generated sitemap partials         │
│  → --omit-build-only removes /dev/astro-test/       │
└─────────────┬───────────────────────────────────────┘
              ↓
┌─────────────────────────────────────────────────────┐
│  astro-cache-bust-postbuild.js                      │
│  → syncs ?v=HASH in dist HTML to match assets       │
│  → uses cache-bust-assets.js as source of truth     │
└─────────────────────────────────────────────────────┘
```

**Key insight:** The legacy sitemap.xml OVERRIDES Astro's sitemap generation. This means new Astro pages won't appear in the sitemap until someone manually updates the committed sitemap.xml. This is a deliberate architectural choice ("Until Astro owns sitemap generation for the full site") but creates a maintenance gap.
