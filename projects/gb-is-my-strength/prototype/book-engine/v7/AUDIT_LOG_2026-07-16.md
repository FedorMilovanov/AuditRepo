# Аудит v7: настройки чтения и запрет гибридного UI

**Дата:** 16 июля 2026

## Вывод

Экран settings из прототипа — **не точный экран актуального Gill-движка**.

У production и prototype совпадает только назначение desktop-позиции: настройки открываются компактным popover от шестерёнки внизу-слева rail, без затемнения страницы. Внутренний DOM, контролы, состояния и runtime-контракты у prototype были отдельными (`.set-*`), хотя research package прямо запрещает переносить их буквально.

Исправление применено: `.set-*` удалён из production-facing прототипа и заменён адаптером текущего контракта `GillReaderSettingsSheet.astro` — `gillSettingsOverlay`, `gill-settings-sheet`, сегменты с `aria-pressed`, font-track с пятью точками и синхронизация `aria-expanded` обоих триггеров.

## Интеграционное правило

1. **Book mode** — единственная новая функция: hierarchy `book → chapter → article → H2/H3`.
2. Play, search, save, settings, menu, mobile chrome и reader controls должны использовать существующие source contracts.
3. Если standalone-prototype обязан показать production control, он копирует source markup/classes/runtime semantics с минимальным adapter-кодом. Он не создаёт параллельный визуальный язык.
4. Любая новая визуальная правка требует повторить матрицу выше и добавить запись в этот файл.

## Follow-up contract corrections

- Current `origin/main` overrides stale `PremiumControls` documents. The standalone book prototype now follows the exact current Gill corner contract: 32px `gbs-rail-ember`, five speeds (`1×…2×`) and the minimal-number desktop exception.
- Prologue and Reference now use the production `GillLeatherRibbon.astro` SVG structure and `GillLeatherDefs.astro` filters; the handmade `.ribbon` implementation is removed.
- Series progress is derived through a local equivalent of source `heartProgress()` over current `HEART_SERIES_ITEMS`; for the first article it yields `done=39`, `part=41`, `total=228`. No hand-entered total remains.
- Regression caught by the post-port runtime audit: the prototype initially read obsolete fields `completedMinutes/partMinutes/totalMinutes` from the source-shaped `{ doneMin, partMin, totalMin }` object, yielding `NaN%`. It now reads the exact source names; 0% article progress produces 17% book progress, 42.5% produces 25%, 100% produces 35%.
