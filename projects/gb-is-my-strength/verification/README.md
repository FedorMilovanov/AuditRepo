# Verification

Эта папка хранит документы, где несколько intake-потоков уже **сверяются между собой**.

Разница с `working/`:
- `working/` — это любые промежуточные synthesis/triage notes;
- `verification/` — это уже cross-reference между несколькими агентами или intake-волнами.

Типичные документы здесь:
- cross-reference synthesis
- duplicate-finding reconciliation
- confirmed vs disputed split
- unified bug taxonomy before final move into `verified/`
