## 5. Правила закрытия (неизменны)

1. Закрытие только с: deterministic failing fixture → fix → source+dist+browser гейты → live-проверка (если delivery) → immutable SHA в AuditRepo.
2. Запрещённые основания закрытия: «build passed», «visual looked correct», «legacy и Astro совпадают», зелёный шаг workflow (для IndexNow/deploy — только асерт HTTP/идентичности).
3. `[skip ci]`-HEAD не считается доказанным без независимого прогона.
4. Паритет ≠ правда; parity/schema/grounding/source-truth — четыре разных проверки.
5. Позитивные заявления — только с именованным инвариантом, окружением и негативным тестом.
6. Минимальный релизный командный контракт и acceptance — V3 §17–18 (обязателен при закрытии W1–W4).

## 7. Как закрывать ЭТОТ аудит

Этот документ — единственная «текущая правда» расширенного бэклога. Закрывать волнами W1→W10:
на каждый закрытый ID — строка в MASTER_BUG_MATRIX (перенос из этого дока с SHA фикса),
статус здесь менять на CLOSED с witness. Когда все волны закрыты — документ архивируется
целиком и заводится новое исследование с нового HEAD. Не плодить параллельных
«current»-документов: новые находки — в матрицу и/или сюда через reverify-процедуру.
