## NEW-3 — Nagornaya font buttons dead (P1)

### Confirmed bug (double-confirmed: arena-agent-2 + verifier-2)
- **Markup** (`nagornaya/chast-1..5/index.html`):
  `<button class="nag-fontsize-btn" id="nagFontDec" ...>` / `id="nagFontInc"`
- **JS** (`js/nagornaya-mobile-toc.js`) listens for:
  `querySelector('[data-fontsize="down"], .nag-fontsize-down')` /
  `querySelector('[data-fontsize="up"],   .nag-fontsize-up')`
- → 0 matching elements → A−/A+ do nothing.

### Validated fix (JS-side, backward-compatible, 1 file)
Add the actual IDs to the existing selector list (keeps the old selectors working):

```diff
--- a/js/nagornaya-mobile-toc.js
+++ b/js/nagornaya-mobile-toc.js
- var a=document.querySelector('[data-fontsize="down"], .nag-fontsize-down'),
-     r=document.querySelector('[data-fontsize="up"],   .nag-fontsize-up');
+ var a=document.querySelector('[data-fontsize="down"], .nag-fontsize-down, #nagFontDec'),
+     r=document.querySelector('[data-fontsize="up"],   .nag-fontsize-up,   #nagFontInc');
```

This restores the font-size controls on all 5 Nagornaya pages. Independent of PS-01.

> Alternative (markup-side): add `data-fontsize="down"`/`data-fontsize="up"` to the
> buttons in `*PageChrome.astro`. The JS-side fix is preferred (1 file vs 5 pages).

## NEW-5 — feed.xml wrong weekdays (P2)

### Confirmed bug (double-confirmed: arena-agent-2 + verifier-2)
9 `<pubDate>` entries have an RFC-822 weekday that disagrees with the date:
- 3× Gill: `Sat, 31 May 2026` → actually **Sun**
- 6× Nagornaya: `Thu, 01 May 2026` → actually **Fri**

### Root cause — the generation function is ALREADY correct
`scripts/update-meta.js` `toRFC()`:
```js
function toRFC(d){return new Date(new Date(d).getTime()+3*3600000).toUTCString().replace('GMT','+0300');}
```
Live check on the date in question:
```
toRFC('2026-05-31') → "Sun, 31 May 2026 03:00:00 +0300"   ✅ correct weekday
```
So the **current code produces correct weekdays**. The stale `feed.xml` was generated by
an older/buggy version (likely one that computed the weekday from local tz or used a
wrong offset). No code change needed.

### Validated fix — regenerate feed.xml
```bash
node scripts/update-meta.js --all    # regenerates pubDate via the now-correct toRFC()
# then verify:
grep -oE '<pubDate>[^<]+</pubDate>' feed.xml
# 2026-05-31 entries should read "Sun", 2026-05-01 should read "Fri"
```
(This also incidentally addresses P2-6 if the `+0000`→`+0300` offset is part of the same
regen; verify separately.)
